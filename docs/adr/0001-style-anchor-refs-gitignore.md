# 스타일 앵커 원본은 gitignore 된 refs/ 에만 두고 어떤 리포에도 커밋하지 않는다

스타일 앵커(생성 호출에 `style_images` 로 넣는 참조 묶음)의 재료가 **상용 게임 스크린샷/스프라이트**다. 이걸 `library/` 처럼 git 으로 push 되는 곳에 넣는 순간 스타일 참조가 아니라 **재배포**가 되므로, 앵커는 플러그인 리포 안 `refs/<앵커이름>/` 에 두되 **gitignore 로 커밋에서 완전히 배제**한다 (`refs/README.md` 만 예외로 커밋). 그 대가로 앵커는 타 기기와 git 으로 공유되지 않는다 — 이 트레이드오프는 인지하고 감수한 선택이다(2026-07-17, 단일 기기 워크플로 전제).

## Considered Options

- **게임 프로젝트 로컬 + gitignore** — 게임마다 스타일이 달라 자연스러우나, 사용자가 플러그인 리포 한 곳 관리를 선호해 기각.
- **커밋해서 기기 간 공유** — 상용 원본 재배포라 법적으로 불가. 기각.
- **그때그때 임시 전달** — 재현성·세트 일관성이 무너져 기각(세트 스타일 불일치가 해결하려던 문제였음).

## Consequences

- 생성 산출물의 캐시 메타에는 앵커 **이름만** 기록한다(`style.anchor`) — 원본 이미지는 기록하지 않는다.
- 다른 기기에서 같은 앵커를 쓰려면 원본을 수동으로 다시 준비해야 한다.
- 산출물이 특정 상용 게임 캐릭터/에셋의 근사 복제로 나오면 채택하지 않는다(스타일 참조와 복제의 경계).
