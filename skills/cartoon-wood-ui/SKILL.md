# 카툰 우드 UI 킷 적용 플레이북

Planet Blitz 격납고 파일럿(2026-07-19, PR #62~#65)에서 검증된 **카툰 우드 UI 킷**을 다른 화면/게임에 적용할 때의 정본. 킷 자산은 이 리포 `library/` 에 캐시 등록돼 있다 — **재생성 금지, `pixellab-cache find` 로 재사용**한다(pixellab 스킬 §4).

## 1) 킷 구성 (library/ 캐시 id · 9-slice 파라미터)

| 캐시 id | 내용 | 크기 | 사용법 |
|---|---|---|---|
| `pb_ui_wood_kit_sheet` | 패널·타이틀바·버튼 3색·슬롯·툴팁 원시트 | 512² | 슬라이스 좌표: panel(8,8,308,308) titlebar(330,8,504,68) btn r/b/y(330,92·162·232,+150×54) tooltip(330,304,504,504) slot(8/104/200,330,+84²) |
| `pb_ui_wood_blank_buttons` | **무지(텍스트 없음)** 배너+버튼 4색+칩 | 512×256 | banner(462×54) bb_red/blue/yellow/wood(~140×48) bb_chip(187×44) |
| `pb_ui_icon_*` 16종 | close/coin/crystal/star/lock/salvage/expand/arrow_l·r/gear/upgrade/shield/rocket/check/search/trash | 64² | 통짜 스케일 |
| `pb_ship_showcase_fighter` | 기체 쇼케이스 일러스트 | 128² | ×2 nearest 확대 전시 |

- **9-slice 테두리**: 패널 46px 사방 · 배너 좌우 캡 40 · 버튼 좌우 캡 30 · 칩 캡 24. 슬롯/아이콘은 9-slice 불필요.
- **팔레트**: 배경 `#181426`(별 장식 금지 — 사용자 확정), 패널 내부 `#1c182e` 계열 245α, 텍스트 크림 `#ebdcbe`/골드 `#ffd678`/회갈 `#aa9b87`, 연결선 금색 `(255,214,120)` α150, 등급색은 게임 기존 문법 유지.
- 렌더는 전부 `scaleMode='nearest'`. 참조 구현: planet-blitz `src/ui/pixi/` (nineSlicePanel·button·slotGrid·tooltip·titleBar·hangar).

## 2) 적용 순서 (승인 게이트 포함)

1. **캐시 조회** — `pixellab-cache find --anchor pb-cartoon-wood-ui` 로 킷 재사용. 새 부품이 필요할 때만 create_ui_asset 으로 **같은 앵커·같은 팔레트** 배치 생성.
2. **목업 승인 게이트** — 구현 전에 실제 자산을 합성한 1920×1080 목업 1장을 만들어 로컬 http 서버로 사용자 브라우저에 열고 승인받는다. 목업에서 확정된 좌표·폰트·간격이 곧 구현 사양이다(아래 §4-8).
3. **구현** — 목업 좌표를 그대로 상수로 옮긴다. 재해석 금지 — 파일럿에서 "구현자가 좌표를 재유도"한 것이 목업 불일치 5건의 근원이었다.
4. **실브라우저 검증** — 반드시 **1920×1080 뷰포트 스크린샷**을 찍어 목업과 나란히 대조한 뒤에 완료를 선언한다. 테스트/빌드 통과 ≠ 시각 검증.

## 3) 함정 1 — 생성 자산에 구워진 텍스트/장식 (최다 재발)

`create_ui_asset` 은 pieces label 을 그대로 **텍스트로 구워 넣는다**("STATS", "CHARACTER INFO" 등). 구워진 텍스트는 9-slice 중앙 스트레치에서 흉하게 늘어난다.

- **버튼/배너/칩**: 반드시 **무지(blank) 자산**을 쓰고 텍스트는 코드로 렌더한다. 신규 생성 시 label 에 `"blank ..., no text"` 를 명시. (`pb_ui_wood_blank_buttons` 가 이렇게 만든 것.)
- **패널**: 중앙에 장식이 구워졌으면 **중앙을 투명 처리**(테두리 46px 링만 남김)하고, 어두운 내부 채움은 코드로 그린다(프레임보다 먼저 그려 프레임이 덮게).
- 9-slice 중앙은 "테두리 바로 안쪽 얇은(≤6px) 깨끗한 띠"를 늘리거나 아예 투명+코드 채움으로.

## 4) 함정 2 — 프레임 침범 (마스크·행간·라벨)

나무 프레임(46px)은 시각적으로 두꺼워서 콘텐츠가 1px 만 넘어도 티가 난다.

- **스크롤 마스크 하한 = 패널 안쪽 프레임 위** (`py + ph - 46 - 여유4`). 패널 높이에서 대충 빼면 셀이 테두리 위에 얹힌다.
- **마스크 높이는 행 단위로 클램프** — `floor(가용높이/행높이)*행높이`. 반토막 셀·반토막 글자가 보이면 사용자는 "삐져나왔다"고 인지한다.
- **슬롯 바깥쪽 라벨**은 라벨 폭(~90px)만큼 컬럼을 안쪽으로: 컬럼 x ≥ 패널 x + 46 + 라벨폭 + 12.
- **N행 배치는 간격을 역산**: `step = (안쪽하한 - 시작y - 셀크기) / (N-1)`. 고정 간격을 먼저 정하면 마지막 행이 걸친다.

## 5) 함정 3 — 레터박스/DPR (Pixi v8)

- `renderer.width/height` 는 **이미 논리(CSS) 픽셀**이다. `resolution` 으로 다시 나누면 Windows 배율(dpr>1)에서 화면 우/하단이 비고 UI 가 축소된다. `app.screen.width/height` 를 쓴다.
- `resizeTo` 경로의 renderer 'resize' 이벤트가 리스너까지 안 오는 환경이 있다 — **window resize 를 직접 구독**하고 rAF 한 프레임 뒤 재적합한다.
- 검증은 dpr>1 환경 + 창 리사이즈 반복으로.

## 6) 함정 4 — 텍스트 선명도

디자인 스페이스(1920×1080)가 축소 표시되면 기본 해상도 Text 는 흐릿/깨져 보인다. **모든 Pixi Text 에 `resolution: 2`** 를 준다. 폰트는 한글 스택("Malgun Gothic", sans-serif), 본문 24/설명 15 이상 + 행 간격 48 이상(빡빡하면 사용자가 바로 지적한다).

## 7) 함정 5 — DOM/캔버스 레이어 혼재

캔버스 UI 화면은 기존 **DOM HUD(HP/LV 등)를 가리지 못한다**(전면 DOM 오버레이 화면들은 자연히 가렸던 것). 캔버스 화면 show/hide 에서 해당 DOM 을 명시적으로 숨기고 복원한다.

## 8) 목업-구현 정합 체크리스트 (완료 선언 전 필수)

- [ ] 1920×1080 스크린샷 vs 승인 목업 나란히 대조 (배너·패널 간격, 버튼 위치, 폰트 크기)
- [ ] 프레임 침범 0 (슬롯·라벨·버튼·텍스트 전부)
- [ ] 반토막 노출 0 (마스크 행 단위 클램프)
- [ ] dpr>1 + 창 리사이즈에서 레터박스 정상
- [ ] DOM 오버레이(HUD·기어 버튼) 겹침 없음
- [ ] 텍스트 선명도 (resolution 2)
- [ ] 사용자 직접 플레이 판정 (로컬 서버 → 사용자 브라우저)
